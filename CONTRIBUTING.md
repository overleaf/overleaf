Contributing to Lemma
==========================

Thank you for reading this! If you'd like to report a bug or join in the development
of Lemma, then here are some notes on how to do that.

Reporting bugs and opening issues
---------------------------------

If you'd like to report a bug or open an issue, please **[check if there is an existing issue](https://github.com/TryBayes/LemmaForLatex/issues).**
If there is then please add any more information that you have, or give it a üëç.

When submitting an issue please describe the issue as clearly as possible, including how to
reproduce the bug, which situations it appears in, what you expected to happen, and what actually happens.
If you can include a screenshot for front end issues that is very helpful.

**Note**: If you are using [lemmaforlatex.com](https://lemmaforlatex.com) and have a problem, or if you would like to request a new feature, please contact the Support team at founders@lemmaforlatex.com. Raise an issue here only to report bugs in the self-hosted version of Lemma.

Pull Requests
-------------

See [our wiki](https://github.com/TryBayes/LemmaForLatex/wiki)
for how to manage the Lemma development environment and for our developer guidelines.

We love pull requests, so be bold with them! Don't be afraid of going ahead
and changing something, or adding a new feature. We're very happy to work with you
to get your changes merged into Lemma.

If you're looking for something to work on, have a look at the [open issues](https://github.com/TryBayes/LemmaForLatex/issues).

Security
--------

Please see [our security policy](https://github.com/TryBayes/LemmaForLatex/security/policy) if you would like to report a potential security vulnerability.
